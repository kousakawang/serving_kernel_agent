# Task Pack README

This task pack is generated by Framework Engineer for Kernel Engineer.

Kernel Engineer should optimize only the candidate implementation under the ABI defined in `task.yaml`.

## Files

- `task.yaml`: task contract and kernel ABI.
- `snapshots/manifest.json`: selected snapshot cases used for replay.
- `shape_list.json`: summary/index derived from selected snapshots, not the replay source.
- `snapshot_runtime.py`: standalone replay runtime copied into this task pack.
- `env_manifest.yaml`: available implementation/profiling environment.
- `original_impl.py`: original captured target replay used as benchmark baseline.
- `reference_impl.py`: reference entry plus snapshot-golden correctness fallback.
- `candidate_impl.py`: Kernel Engineer implementation entry.
- `correctness_test.py`: correctness harness. Do not relax tolerance.
- `benchmark.py`: benchmark harness. Do not change timing rules.
- `scripts/run_correctness.sh`: correctness command.
- `scripts/run_benchmark.sh`: benchmark command.
- `scripts/run_ncu.sh`: NCU command for hot cases.

## Allowed Edits

Kernel Engineer may edit:

- `candidate_impl.py`
- `kernel_sources/`
- `docs/iteration_log.md`
- generated benchmark/profiler reports

Kernel Engineer must not edit:

- `snapshots/`
- `snapshot_runtime.py`
- `shape_list.json`
- `original_impl.py`
- `reference_impl.py`
- `correctness_test.py`
- `benchmark.py`
- `task.yaml`, unless requesting a FrameworkChangeRequest

## Quick Start

```bash
bash scripts/run_correctness.sh
bash scripts/run_benchmark.sh
```

If either command fails before candidate work begins, return a task acceptance review to Framework Engineer.
